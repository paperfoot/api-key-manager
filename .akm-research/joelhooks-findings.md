# joelhooks/agent-secrets findings

## What it is

A Go single-binary CLI (`secrets`) backed by a long-running Unix-socket daemon (`secrets serve`) that stores secrets in an Age-encrypted file under `~/.agent-secrets/`. Agents request a time-bounded "lease" (UUID, default 1h, max 24h) over JSON-RPC instead of reading values directly — the daemon refuses `secrets.get` with `RPCUnauthorized` (`internal/daemon/handlers.go:67-72`). Command surface: `init`, `add`, `update`, `delete`/`rm`, `list`, `lease`, `revoke [id|--all]`, `audit`, `status`, `health`, `scan`, `env`, `exec`, `cleanup`, `serve`, `self-update` (`cmd/secrets/root.go:46-62`). All responses are JSON envelopes with `ok`/`success`/`next_actions`/`fix` except `lease`, which returns the raw value to stdout for shell substitution (`cmd/secrets/lease.go:91-95`). Cross-platform (Linux/macOS/WSL) — no macOS-specific dependency.

## Backing store

A single Age-encrypted JSON blob at `~/.agent-secrets/secrets.age`, encrypted to an X25519 recipient whose private key sits in `~/.agent-secrets/identity.age`. Identity is generated via `age.GenerateX25519Identity()` on `secrets init`, written with `0600` (`internal/store/encryption.go:14-30`). Encryption uses the standard `filippo.io/age` streaming `Encrypt`/`Decrypt` against the recipient (`internal/store/encryption.go:52-91`). Plaintext payload schema is `{version:1, secrets:{name:{Secret, value}}}` (`internal/store/store.go:17-26`). The store decrypts the whole blob into memory on daemon start (`internal/store/store.go:133-138`) and never persists plaintext to disk. Directory is `0700`, all key files `0600`, validated on every `Load` via `ValidateAllKeyFiles` (`internal/store/permissions.go:39-82`) — a "world-readable" identity file aborts startup with a `chmod 600` fix string. Leases persist as a plaintext JSON file `leases.json` at `0600` (`internal/lease/manager.go:286-307`); audit log `audit.log` is JSONL `0600` append-only with `fsync` after every write (`internal/audit/audit.go:25, 37-54`). Socket also `chmod 0600` (`internal/daemon/daemon.go:128-133`). No keychain or OS keystore — the on-disk Age key is the root of trust.

## Session leases

A "lease" is an in-memory + on-disk record `{ID uuid, SecretName, ClientID, CreatedAt, ExpiresAt, Revoked}` (`internal/types/types.go:18-25`). Granted by `Manager.Acquire(name, clientID, ttl)` which validates `ttl <= MaxLeaseTTL` (default 24h, `internal/lease/manager.go:62-70`) and falls back to `DefaultLeaseTTL` (1h) when unset. Dedup: an existing active lease for the same `(clientID, secretName)` is silently revoked and replaced (`internal/lease/manager.go:84-93`) — prevents accumulation. `client_id` defaults to the hostname if the caller doesn't pass `--client-id` (`cmd/secrets/lease.go:45-52`), so audit attribution for a solo dev is just "this machine".

**Critical caveat: the lease never gates anything after issuance.** The lease record is created at the same time as the value is returned to stdout (`internal/daemon/handlers.go:304-321`); once the agent has `$TOKEN` it owns plaintext forever. `lease.Revoke` only flips `Revoked=true` on the record (`internal/lease/manager.go:138`) — it does not, and cannot, recall the exported environment variable already in the child's memory. There is no second-call validation: nothing in the daemon ever consumes a `lease_id` again. The README admits "Leases cannot be extended—must acquire new lease" but the only consumer of `IsValid` is the cleanup loop, the `List` filter, and `health` (`internal/lease/manager.go:213-228`, `internal/daemon/handlers.go:467`).

Expiry: a background goroutine in the daemon ticks every minute and deletes expired records, logging `lease_expire` per lease (`internal/lease/manager.go:262-277`, `internal/daemon/daemon.go:147`). On `secrets delete <name>`, the daemon calls `RevokeBySecret(name)` first (`internal/daemon/handlers.go:230-237`). Verdict: **the lease abstraction is bookkeeping/audit theater for a single-user box** — it logs who asked for what and gives you a list of "still-active" tokens for status pages, but it cannot revoke a value once issued. It only becomes useful in a multi-tenant daemon model that brokers connections (e.g. an HTTP proxy injecting headers per-lease), which this tool does not implement.

## Killswitch

`secrets revoke --all` triggers `Killswitch.Activate` with `RevokeAll=true` (`cmd/secrets/revoke.go:26-31`, `internal/killswitch/killswitch.go:46-52`). Options are a 3-bit struct `KillswitchOptions{RevokeAll, RotateAll, WipeStore}` (`internal/types/types.go:78-82`). What each does:
- `RevokeAll` -> `leaseManager.RevokeAll()` flips every record to `Revoked=true` in-memory and persists (`internal/lease/manager.go:154-173`). Same caveat as above: child processes still hold the values.
- `RotateAll` -> iterates secrets that have a `rotate_via` shell command and runs each through `sh -c <cmd>` with a 30s timeout (`internal/rotation/executor.go:68-77`). Captures stdout+stderr to audit log. Does NOT update the secret value from command output — only calls `store.MarkRotated()` to bump a timestamp (`internal/rotation/executor.go:107`). So `--rotate-via "gh auth refresh"` works (gh writes its own token file) but `--rotate-via "openssl rand -hex 32"` does nothing useful.
- `WipeStore` -> `store.WipeAll()` empties the in-memory map and re-encrypts an empty blob (`internal/store/store.go:316-326`). Identity file remains; only `secrets.age` content is cleared.

Triggers: CLI `revoke --all`, and the optional `HeartbeatMonitor` (`internal/killswitch/heartbeat.go:43-100`). When `config.heartbeat.enabled=true`, a goroutine hits a configured URL every interval; the **first non-2xx response or network error** invokes `killswitch.Activate(config.FailAction)` and then stops the monitor (`internal/killswitch/heartbeat.go:71-92`). Bypass: kill the daemon (anyone with `$USER` can `pkill secrets`), edit `~/.agent-secrets/config.json` to set `heartbeat.enabled=false`, or just don't start the daemon — there is no enforcement layer above process lifecycle.

## Agent integration

Four surfaces (`README.md:412-595`, `skills/secret-management/SKILL.md`):
1. **Raw CLI** for any shell-driving agent: `export TOKEN=$(secrets lease github_token)` (`cmd/secrets/lease.go:93`).
2. **Skill markdown** at `skills/secret-management/SKILL.md` — designed to be copied into `~/.claude/skills/` or `~/.opencode/skills/`. Explicitly mentions Claude Code, Cline, Aider, OpenCode.
3. **OpenClaw plugin manifest** at `openclaw.plugin.json` registering tools `secrets_lease`, `secrets_status`, `secrets_revoke`, `secrets_audit`, `secrets_add`, `secrets_killswitch` — the last two gated behind an explicit agent allowlist (`README.md:534-557`).
4. **Self-documenting root command**: `secrets` with no args prints the full JSON command tree with `next_actions`, intended for an LLM to introspect and chain (`cmd/secrets/root.go:29-72`). Every error envelope carries a `fix` field with a copy-pasteable remediation command (`cmd/secrets/lease.go:73-78`).

No MCP server in this repo. No HTTP API — only Unix socket JSON-RPC for the local CLI.

## Things akm doesn't have (potential adoptions)

- **`secrets exec -- <cmd>` subprocess injection**. Pulls secrets from a configured `.secrets.json` provider, sets them in `env`, runs the child, propagates signals, supports `--ttl` that kills the child after a deadline (`cmd/secrets/exec.go:71-154`). Cleaner than akm's `run --only` because it has no plaintext disk artifact. Effort: low (~150 LOC), value: high.
- **`.secrets.json` project manifest** declaring secret list + env_var names + per-secret TTL + client_id for the whole project, walked up from cwd (`internal/project/config.go:155-184`). Lets a repo declare "this project needs GITHUB_TOKEN and ANTHROPIC_API_KEY" without each agent having to know names. Effort: low, value: high.
- **`.env.local` with TTL metadata headers** (`# secrets-ttl: <RFC3339>`) and a `cleanup` watcher that walks directories and wipes expired env files (`internal/envfile/envfile.go:19-56`, `internal/cleanup/watcher.go:31-94`). Defends against forgotten `.env.local` files surviving the agent session. Effort: low, value: medium.
- **JSON-RPC daemon over Unix socket** (`internal/daemon/daemon.go`, `internal/daemon/handlers.go`). Centralizes lease/audit state across multiple agent processes. Effort: high, value: medium — overkill for solo-dev macOS but the architecture is clean if akm grows.
- **Self-documenting root command tree** with `next_actions` and `fix` fields on every envelope (`cmd/secrets/root.go:29-72`, `output` package). The discoverability-first JSON pattern is genuinely good for agents — `secrets` with no args is a usable replacement for `--help`. Effort: low, value: high.
- **Append-only JSONL audit with `Tail(n)` + `Query(filter)`** including filter by action/secret/time-range (`internal/audit/audit.go:101-143`). akm already has audit; add query-by-action and tail. Effort: low, value: medium.
- **`secrets health` command** flagging expiring leases, never-rotated secrets, and stale secrets (>30 days no access) by replaying audit log (`internal/daemon/handlers.go:447-538`). Effort: medium, value: medium.
- **`secrets scan` for pre-commit-style hardcoded-secret detection** with severity, partial redaction (`first4 + **** + last4`), and exclude patterns (`internal/scanner/scanner.go`, `cmd/secrets/scan.go:96-103`). akm has `guard install/scan` already — this implementation has a cleaner pattern API. Effort: low (already overlapping), value: low.
- **`rotate_via` hook** stored per-secret, executed on killswitch or on demand (`internal/rotation/executor.go:35-116`). Genuinely useful if it actually captured stdout into the new value (currently doesn't — see footguns). Effort: medium, value: medium.
- **Heartbeat killswitch** monitoring a remote URL (`internal/killswitch/heartbeat.go`). For a paranoid "dead-man switch" — agent server dies, all leases die. Effort: medium, value: low (theater for solo dev).

## Things akm does better

- **Keychain over on-disk Age key.** akm uses the macOS Login Keychain (Touch ID / iCloud-backed); joelhooks puts the X25519 private key in `~/.agent-secrets/identity.age` at `0600`. Anyone who reads that file (root, backup snapshot, another process running as `$USER`, a misconfigured Time Machine) gets every secret. akm's keychain item requires unlock and is not in tarball backups.
- **No daemon dependency.** akm runs as a one-shot per command; joelhooks requires `secrets serve &` to be alive for nearly every command. README explicitly warns "if you see 'daemon not running' errors, run `secrets serve &`" (`README.md:85`). Daemon liveness is a real footgun in CI and after reboots.
- **Child stdout/stderr redaction.** akm's `run --only` redacts known values from child output; joelhooks's `exec` does not — a child that `echo $GITHUB_TOKEN` writes the token to the agent's transcript.
- **Push to vercel/gh/fly.** akm has push adapters; joelhooks only has a *pull* from Vercel (`internal/adapters/vercel/vercel.go`).
- **Pre-commit guard install.** akm bundles `guard install` to wire git hooks; joelhooks has a scanner but no hook installer.
- **Typed errors with semantic exit codes.** Both have these; akm's are tighter because there's no JSON-RPC layer to flatten error codes (joelhooks loses specificity at the `RPCError` boundary — `cmd/secrets/lease.go:72-78` parses error messages with `strings.Contains` to recover the fix string).
- **Zero prompts ever.** joelhooks's `secrets add <name>` falls back to an interactive password prompt via `golang.org/x/term.ReadPassword` (`cmd/secrets/add.go:48-57`) when stdin isn't piped — an agent that forgets to pipe will hang.

## Footguns / bugs / weak spots

- **Lease revocation is purely advisory.** Once `LeaseResult.Value` is on the wire, the killswitch cannot recall it from the agent's env or memory (`internal/daemon/handlers.go:315-321`). The threat model in README ("compromised agent leaks all credentials") is *not* mitigated by leases — only by the fact that you only added a few secrets to the store. Theater.
- **No Unix-socket peer credential check.** `daemon.handleConnection` accepts any connection on the socket and never calls `unix.Getsockopt(..., SO_PEERCRED)` (`internal/daemon/daemon.go:233-316`). `0600` socket perms keep other UIDs out, but any process running as `$USER` (including a sandbox escape, a VS Code extension, an MCP server) can lease every secret with no further check.
- **`rotate_via` does not capture rotated value.** `executor.Rotate` runs the command, logs stdout+stderr, calls `store.MarkRotated()` to bump a timestamp, but never updates `secret.Value` from output (`internal/rotation/executor.go:77-115`). So `--rotate-via "openssl rand -hex 32"` rotates exactly nothing. Only works for hooks that side-effect their own credential file (gh, aws sts).
- **Heartbeat fail = single-shot wipe.** First non-2xx response wipes/revokes per config and the monitor stops itself (`internal/killswitch/heartbeat.go:88-92`). One transient network blip can wipe the store if `WipeStore=true`. No retry, no debounce, no quorum.
- **`secrets add` requires a daemon.** Adding a secret goes over JSON-RPC (`cmd/secrets/add.go:76`). If the daemon isn't running, you can't even seed initial secrets — the install flow needs `secrets serve &` first.
- **`leases.json` plaintext.** Lease metadata includes `client_id` and `secret_name` — disclosed if the file leaks. Not catastrophic (no values), but tells an attacker which secrets to target.
- **No process binding.** A lease isn't tied to a PID/PGID/socket-connection. Acquire-and-die leaves the lease "active" for the full TTL with no consumer, polluting `status` and `health` (`internal/lease/manager.go:213-228`).
- **Trailing whitespace stripped from secrets.** `cmd/secrets/add.go:60` does `value = strings.TrimSpace(value)` — silently corrupts secrets that legitimately have trailing whitespace (rare but real for some PEM keys).
- **`secrets exec` re-fetches every invocation** via the Vercel adapter (`cmd/secrets/exec.go:53`) — network round-trip on every run, no caching, no offline mode.

## Verdict

Adopt three things, in priority order: (1) the **`.secrets.json` project manifest + `secrets exec -- <cmd>` subprocess injection** pattern (`internal/project/config.go`, `cmd/secrets/exec.go`) — this is the single highest-leverage idea in the repo, lets a project declare its credential dependencies without each agent reinventing names, and replaces ad-hoc shell exports with a clean lifecycle; (2) the **self-documenting root command with `next_actions` and `fix` envelopes** (`cmd/secrets/root.go:29-72`, `cmd/secrets/lease.go:99-121`) — concrete remediation strings on every error class measurably improve agent recovery from misuse; (3) the **`.env.local` TTL header + `cleanup` watcher** (`internal/envfile/envfile.go`, `internal/cleanup/watcher.go`) — defends against forgotten env files post-session, complements akm's existing in-memory injection. Skip the lease/killswitch model itself: against an in-process compromise it's audit theater, and akm's keychain-resident keys already give a stronger at-rest story than this tool's on-disk Age key.
